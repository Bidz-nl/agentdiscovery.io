Windsurf briefing — ADP 2.0 Handshake

We bouwen in ADP 2.0 een nieuwe verplichte laag tussen Register en Discover:

Nieuwe flow:
Register → Handshake → Discover → Negotiate → Transact → Reputation

Waarom

ADP v0.1 doet al discovery, matching, negotiation en transacties, en positioneert zich als commerce layer voor agents. Maar er ontbreekt nog een formele, automatische manier waarop twee agents elkaar eerst herkennen, protocolversie afspreken, authority samenvatten, trust-context opbouwen en een sessie openen voordat discovery/negotiation begint. Dat is de ontbrekende “hand”.

Doel van de handshake

De handshake moet automatisch regelen:

identity confirmation

protocol version negotiation

authority summary exchange

trust bootstrap

session creation

readiness for discovery / negotiation

Productbeslissing
Nieuw concept

Business rules
Regel 1

Discovery en negotiation mogen in v2.0 alleen starten als er een geldige handshake session is.

Regel 2

Bij incompatibele protocolversies mag fallback naar v1.0 alleen als beide agents dat ondersteunen.

Regel 3

Trust level start standaard als provisional en kan later omhoog na verificatie of eerdere transacties.

Regel 4

Authority boundaries worden niet volledig gedeeld, maar als samenvatting of digest om privacy en security te bewaren.

Regel 5

Handshake sessions zijn kort geldig, bijvoorbeeld 5 tot 15 minuten TTL.

import { v4 as uuid } from "uuid"

export function createHandshakeSession(request) {

  const session = {
    session_id: "hs_" + uuid(),
    initiator_did: request.did,
    protocol_version: request.protocol_version,
    status: "open",
    trust_level: "provisional",
    created_at: new Date().toISOString(),
    expires_at: new Date(Date.now() + 15 * 60 * 1000).toISOString()
  }

  return session
}

import { createHandshakeSession } from "./handshake.service"

export async function handshake(req, res) {

  const session = createHandshakeSession(req.body)

  res.json({
    message_type: "ACK",
    accepted: true,
    session_id: session.session_id,
    trust_level: session.trust_level,
    status: session.status,
    expires_at: session.expires_at
  })
}
