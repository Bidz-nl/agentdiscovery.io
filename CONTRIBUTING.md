# Contributing

Thanks for your interest in contributing to the Agent Discovery Protocol (ADP) repository.

## Reporting issues

If you find a bug, inconsistency, or documentation problem, please open an issue with:

- a short summary
- clear reproduction steps, if relevant
- expected behavior
- actual behavior
- any useful logs or screenshots

For protocol-related issues, describe whether the problem affects protocol semantics, route behavior, documentation, or developer experience.

## Submitting pull requests

Please keep pull requests focused and easy to review.

Recommended flow:

1. Fork the repository and create a focused branch
2. Make a small, clearly scoped change
3. Verify that your change does not unintentionally affect unrelated behavior
4. Open a pull request with a concise explanation of what changed and why

If your change touches protocol-facing behavior, explain how it aligns with the ADP v2 flow and existing documentation.

## Code style expectations

Please aim for:

- small, focused changes
- clear naming
- consistency with existing file structure and response conventions
- no unrelated refactors bundled into the same pull request

For documentation changes, prefer short, precise, developer-friendly language.

## Staying aligned with ADP scope

This repository contains an ADP v2 MVP protocol implementation, documentation set, and demo materials.

When contributing, please keep changes aligned with the current ADP protocol scope:

- avoid adding unrelated features without discussion
- avoid changing protocol behavior casually
- avoid architectural refactors unless clearly justified and discussed first
- prefer incremental improvements that preserve protocol clarity and consistency

If you want to propose larger protocol changes, open an issue first so the scope and rationale can be discussed before implementation.
